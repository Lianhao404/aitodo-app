# AI辅助编程与工程化实训个人总结报告

## 一、项目概述

本次居家实训我独立完成了一个基于 Next.js + Flask 的待办事项管理系统。项目采用前后端分离架构，前端使用 Next.js 14 框架开发三个页面（待办列表页、添加待办页、关于页），后端使用 Flask 框架提供三个 RESTful API 接口（获取待办列表、新增待办、删除待办），数据库采用轻量级的 SQLite。

## 二、遇到的最大困难及解决过程

开发过程中遇到的最大困难是后端数据库初始化的问题。最开始我把建表函数 init_db() 放在了 if __name__ == '__main__' 代码块中，用 python app.py 方式启动没问题，但改用 flask run 命令启动时，数据库表一直创建失败，一直报错 "no such table: todos"。

一开始我以为是 SQL 语句写错了，反复检查建表代码都没发现问题。后来通过 AI 辅助才明白原因：flask run 启动方式不会执行 if __name__ == '__main__' 里面的代码，所以 init_db() 从来没被调用过。解决方法是把 init_db() 调用移到模块顶层，在 Flask 应用创建后就立即执行，这样无论哪种启动方式都能自动建表。

另一个印象深刻的问题是跨域问题。前端页面运行在 3000 端口，后端运行在 5000 端口，浏览器出于安全策略会阻止跨域请求。解决方法是后端安装 flask-cors 插件，一行 CORS(app) 就搞定了，让我对前后端分离架构的跨域问题有了直观认识。

## 三、对 AI 辅助编程的心得体会

这次实训让我深刻体会到了 AI 辅助编程的效率优势。

首先，AI 能快速生成基础代码骨架。比如 Flask 的三个接口、Next.js 的页面组件，这些有固定模式的代码，AI 几秒钟就能生成一个可运行的版本，比自己从零开始写节省了大量时间。

其次，AI 是很好的排错助手。遇到报错时，把错误信息发给 AI，它能快速定位问题原因并给出解决方案，比自己瞎猜或者搜 Stack Overflow 高效很多。比如数据库表不存在的问题，自己可能要卡很久，AI 一句话就点醒了我。

但 AI 也不是万能的。生成的代码有时会有小问题，需要自己仔细检查和调试。而且如果自己完全不懂代码逻辑，AI 生成的东西出了 bug 也不知道怎么改。所以 AI 更像是一个"高效的助手"，而不是"替代程序员的工具"。开发者还是要理解代码逻辑，才能用好 AI。

## 四、个人在项目中的具体贡献

本次项目为个人独立完成，所有工作均由本人完成：

1.项目架构设计：确定前后端分离技术选型，规划项目目录结构 

2.后端开发：编写 Flask 三个 API 接口，设计 SQLite 数据库表结构，解决跨域问题 

3.前端开发：使用 Next.js 开发三个页面，实现前后端数据联调 

4.工程规范：使用 Git 进行版本管理，保持规范的 Commit Message，分多天提交代码 

5.文档编写：撰写 README 项目文档、Prompt 日志、API 接口说明 

6.问题排查：独立解决数据库初始化、跨域等开发中遇到的各类问题 

## 五、总结与收获

通过这次七天居家实训，我完整地走了一遍全栈项目开发流程，从环境搭建、代码编写到联调测试，对前后端分离开发有了更深入的理解。同时也熟练掌握了 Git 版本控制的工作流，明白了"过程留痕"在团队协作中的重要性。

AI 辅助编程大大提升了开发效率，但也让我明白，扎实的编程基础和问题排查能力才是根本。AI 可以帮你写代码，但理解代码、调试代码、把控整体架构的能力，还是要靠自己不断积累。

未来我会继续学习工程化相关知识，比如 CI/CD、单元测试、容器化部署等，让自己的项目更加规范和专业。
